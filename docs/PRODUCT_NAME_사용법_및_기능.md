## 1. PRODUCT_NAME 개요

### 1.1 제품 한눈에 보기
PRODUCT_NAME(ModelHub)는 AI 모델과 학습 데이터셋을 한곳에서 찾아보고, 구매하고, 바로 실행까지 할 수 있도록 설계된 웹 기반 통합 플랫폼입니다. 모델 제작자와 구매자, 그리고 운영 관리자가 동일한 환경에서 협업하며 지식과 리소스를 거래할 수 있도록 지원합니다. 단순한 마켓플레이스를 넘어, 실험 공간(플레이그라운드)과 파인튜닝 도구, 청구·정산 관리 도구까지 제공하여 AI 프로젝트의 모든 과정을 연결해 줍니다.

### 1.2 핵심 가치
1. **탐색과 비교의 편의성** – 고급 필터와 실시간 비교 기능으로 원하는 모델을 빠르게 찾을 수 있습니다.
2. **즉시 사용 가능한 실험 환경** – 별도 IDE 없이도 브라우저에서 모델을 테스트하고 파라미터를 조정할 수 있습니다.
3. **엔드투엔드 운영 지원** – 파인튜닝, 배포, 청구, 모니터링까지 하나의 계정으로 관리합니다.
4. **신뢰 기반 거래** – 스마트 컨트랙트와 지갑 연동으로 결제 및 소유권 이력을 투명하게 확인할 수 있습니다.

### 1.3 대상 사용자와 사용 시나리오
- **모델 구매자:** 프로젝트 요구에 맞는 모델을 찾고, 시험 사용 후 구매 및 배포까지 진행.
- **데이터 과학자:** 데이터셋을 수집·정제해 모델을 파인튜닝하고 결과를 공유.
- **크리에이터:** 자신이 제작한 모델/데이터를 등록해 판매하고, 고객 피드백을 관리.
- **운영 관리자:** 사용자 권한을 관리하고, 서비스 운영 지표와 감사 로그를 모니터링.

### 1.4 시스템 요구사항
- **하드웨어:** 최소 4GB RAM, 듀얼 코어 CPU 권장. GPU는 필수 아님.
- **네트워크:** 20 Mbps 이상의 안정적인 인터넷 연결. WebSocket 포트를 차단하지 않는 환경 권장.
- **브라우저:** 최신 Chrome, Edge, Firefox, Safari. 모바일 브라우저는 조회 중심으로만 최적화.
- **추가 구성:** 쿠키 및 로컬 스토리지 활성화, 팝업 차단 해제(지갑 연동 시 필요).

### 1.5 지원 브라우저 및 접근성
모든 주요 데스크톱 브라우저 최신 버전을 지원하며, 키보드 내비게이션과 스크린리더 친화적인 ARIA 레이블을 제공하도록 설계되어 있습니다. 색 대비는 WCAG 2.1 AA 기준 이상을 유지하며, 중요한 상태 메시지는 스크린리더가 읽을 수 있도록 `role="alert"`로 표시됩니다.

## 2. 시작하기

### 2.1 계정 생성과 로그인
1. **이메일 가입** – 로그인 페이지에서 이메일과 비밀번호를 입력하면 즉시 계정이 생성되고 확인 메일이 발송됩니다. _결과:_ 첫 로그인 시 온보딩 투어가 시작됩니다. _예시:_ `user@example.com`을 등록하면 "ModelHub에 오신 것을 환영합니다" 이메일 수신.
2. **소셜 로그인** – `Google로 계속하기` 버튼을 클릭하면 OAuth 창이 열리고, 인증 완료 후 자동으로 대시보드로 이동합니다. _결과:_ 조직 정책이 Google 계정에 연동됩니다. _예시:_ 회사 Workspace 계정으로 로그인해 조직별 구매 제한이 자동 적용.
3. **지갑 연동 로그인** – Phantom 지갑 팝업에서 서명을 승인하면 세션이 생성됩니다. _결과:_ 결제 및 소유권 관련 기능이 즉시 활성화됩니다. _예시:_ Phantom 지갑 승인 후 모델 구매 버튼이 활성화.

**로그인 문제 해결 가이드**
- 잘못된 비밀번호 입력 시 "로그인에 실패했습니다" 경고와 함께 재설정 링크가 표시됩니다.
- 2단계 인증이 활성화된 계정은 이메일로 전달된 6자리 코드를 입력해야 세션이 열립니다.
- 네트워크 오류 시 "연결이 원활하지 않습니다" 메시지를 확인하고, `재시도` 버튼으로 API 호출을 다시 수행합니다.

### 2.2 첫 방문 온보딩 요약
온보딩 투어는 5개 핵심 개념을 순서대로 안내합니다.
1. **모델 마켓 탐색** – 카테고리와 태그 필터, 정렬 옵션을 활용해 모델을 찾는 방법.
2. **지갑 연결** – 디지털 자산 지갑과 연동하여 결제와 구매 이력을 관리하는 절차.
3. **플레이그라운드 체험** – 프롬프트 입력과 응답 확인, 비용 추적 인터페이스 안내.
4. **파인튜닝 마법사** – 데이터 업로드, 하이퍼파라미터 설정, 진행률 모니터링 소개.
5. **청구 현황 확인** – 사용량, 청구서, 결제 수단을 관리하는 대시보드 투어.

각 단계는 `다음` 버튼 혹은 `건너뛰기` 링크로 이동할 수 있으며, 툴팁에 단축키와 접근성 정보가 표기됩니다.

### 2.3 역할과 권한 체계

| 역할 | 주요 업무 | 기본 권한 | 추가 권한 부여 조건 | 비고 |
| --- | --- | --- | --- | --- |
| 방문자 | 공개 목록 열람 | 읽기 | 없음 | 로그인 없이 접근 가능 |
| 일반 사용자 | 모델/데이터 탐색, 플레이그라운드 실행 | 읽기 + 실행 | 이메일 인증 | 프로젝트 초급자 대상 |

## 3. 화면과 기능 설명

### 3.1 모델 마켓
- **목적:** 다양한 AI 모델을 탐색하고 비교하여 구매 또는 테스트로 이어지는 사용자 여정을 제공.
- **접근 경로:** 좌측 사이드바 `모델` 메뉴, 홈 대시보드 추천 카드, 직접 URL(`/models`).
- **주요 UI 구성:**
  - 상단 검색창과 정렬 셀렉터(가격, 최신, 인기, 정확도).
  - 좌측 필터 패널(모달리티, 라이선스, 가격 범위 슬라이더, 지원 언어, GPU 호환성 토글).
  - 모델 카드: 썸네일, 태그, 핵심 지표, 즐겨찾기, 비교 체크박스.
  - 하단 비교 도킹바: 최대 3개 모델을 나란히 비교하는 오버레이.
  - 우측 하단 접근성 도우미(키보드 단축키 안내, 화면 낭독자 팁).
- **사용 절차:**
  1. 키워드를 입력하거나 필터를 조합한다 → 검색 결과가 비동기로 갱신 → _예시:_ "text summarization" 입력 후 한국어 지원 필터 ON.
  2. 비교할 모델을 체크하고 `비교하기`를 클릭한다 → 비교 오버레이에 파라미터·성능 지표가 나열 → _예시:_ 두 개의 GPT 계열 모델을 정확도/지연 시간으로 비교.
  3. 마음에 드는 모델을 클릭해 상세 페이지로 이동한다 → 장바구니 및 플레이그라운드 진입 링크 확인 → _예시:_ "Summarizer Pro" 상세 보기.
- **권장 사용 패턴:** 즐겨찾기를 활용해 자주 사용하는 모델을 저장하고, `추천 구성` 배지를 참고하여 평가가 높은 모델을 우선 검토합니다.
- **오류 및 해결:**
  - API 응답 지연 → 우측 상단 로딩 인디케이터가 5초 이상 지속되면 `새로 고침` 버튼으로 재요청.
  - 필터 결과 없음 → 필터 패널 하단 `필터 초기화` 클릭.
- **보안 및 권한 주의:** 결제 기능은 MFA 활성화 및 지갑 연동 후에만 접근할 수 있으며, 가격은 Lamports로 표시되므로 환율 위젯을 참고합니다.
- **관련 링크:** 모델 상세, 장바구니, 추천 모델 가이드.

### 3.2 모델 상세
- **목적:** 모델의 정확도, 활용 사례, 라이선스, 업데이트 이력 등을 검토하고 구매 또는 플레이그라운드 실행을 결정하도록 지원.
- **접근 경로:** 모델 마켓에서 카드 클릭, 즐겨찾기 목록, 직접 URL(`/model/{id}`).
- **주요 UI 구성:** 상단 요약 헤더(평점, 다운로드 수, 최신 버전), 가격 플랜 탭, 샘플 출력 코드, 기술 사양 표, 무결성 해시, 관련 모델 추천.
- **사용 절차:**
  1. 상단 요약을 검토한다 → 모델 버전과 핵심 메트릭을 확인하여 적합성 판단 → _예시:_ MMLU 85%, 응답 지연 800ms.
  2. 가격 플랜을 선택하고 `구매하기`를 클릭한다 → 체크아웃 페이지로 이동하여 결제 수단을 확인 → _예시:_ 표준 플랜(Lamports 120) 선택.
  3. `플레이그라운드에서 테스트` 버튼을 눌러 즉시 프롬프트를 실행한다 → 동일한 설정이 플레이그라운드로 전달 → _예시:_ FAQ 요약 프롬프트 실행.
- **권장 사용 패턴:** `샘플 코드 복사` 버튼으로 API 요청 예시를 가져와 개발 환경에서 바로 테스트합니다.
- **오류 및 해결:** 모델이 비공개 전환되면 "요청한 모델을 찾을 수 없습니다" 메시지가 표시되며, `목록으로 돌아가기`를 통해 다른 모델을 탐색합니다.
- **보안 및 권한 주의:** 지갑 서명이 완료되지 않으면 결제 버튼이 비활성화되며, 무결성 정보(CID, 온체인 트랜잭션 ID)를 통해 모델 파일 변조 여부를 검증할 수 있습니다.
- **관련 링크:** 파인튜닝 마법사, 플레이그라운드, 크리에이터 문의.
- ![스크린샷: 모델 상세](path/to/image.png "가격 플랜 및 메트릭")

### 3.3 모델 등록
- **목적:** 제작자가 신규 모델을 등록하고 검수 요청을 제출.
- **접근 경로:** 헤더 `모델 등록`, 크리에이터 허브 `새 모델 등록`, URL(`/models/register`).
- **주요 UI 구성:** 단계별 진행 표시줄, 기본 정보 탭, 버전 관리 탭, 가격 및 라이선스 설정, 미디어 업로드(썸네일/샘플 코드), 검수 체크리스트.
- **사용 절차:**
  1. 모델 기본 정보와 태그를 입력하고 저장한다 → 초안 상태로 임시 저장 → _예시:_ "Korean LLM" 설명과 `korean`, `enterprise` 태그 추가.
  2. 버전 번호와 릴리스 노트를 작성한 후 파일을 업로드한다 → 업로드 완료 시 해시가 자동 생성 → _예시:_ v1.0 모델 패키지 업로드.
  3. 가격과 라이선스 정책을 설정하고 `검수 요청` 버튼을 클릭한다 → 검수 대기 상태로 전환되며 알림이 발송 → _예시:_ 연구용 무료, 상업용 유료.
- **권장 사용 패턴:** 검수 체크리스트를 다운로드해 데이터 품질과 보안 요건을 사전에 점검한 뒤 제출합니다.
- **오류 및 해결:** 필수 필드가 비어 있으면 붉은 경고와 함께 저장이 차단되며, `필수 항목 보기` 링크가 해당 입력란으로 스크롤합니다.
- **보안 및 권한 주의:** 업로드 파일은 IPFS로 전달되며, 민감한 개인정보는 POLICIES에서 정의한 규칙에 따라 암호화 또는 마스킹해야 합니다.
- **관련 링크:** 크리에이터 허브, 파인튜닝 결과 등록, 제작 가이드라인.
- ![스크린샷: 모델 등록](path/to/image.png "단계별 등록 폼")

### 3.4 데이터셋 마켓
- **목적:** 학습 데이터셋을 탐색·필터링하고 구매 또는 다운로드.
- **접근 경로:** 사이드바 `데이터셋`, 홈 대시보드 추천 카드, URL(`/datasets`).
- **주요 UI 구성:** 상단 검색, 필터 패널(도메인, 포맷, 라이선스, 가격 범위, 데이터 크기), 카드 목록, 빠른 미리보기 모달.
- **사용 절차:**
  1. 필터에서 카테고리와 라이선스를 선택한다 → 결과가 즉시 갱신 → _예시:_ `의료`, `연구용` 조합으로 의료 연구용 데이터셋 조회.
  2. 데이터셋 카드를 클릭해 상세 페이지로 이동하거나 `장바구니` 아이콘으로 바로 결제에 추가한다 → _예시:_ "Medical Image Dataset" 상세 보기.
  3. `빠른 미리보기`를 열어 샘플 데이터를 확인한 후 구매를 확정한다 → _예시:_ JSONL 샘플 10줄 확인.
- **권장 사용 패턴:** 무료 데이터셋으로 먼저 모델을 시험한 뒤, 유료 데이터셋을 구매해 성능을 향상시키는 단계적 접근이 추천됩니다.
- **오류 및 해결:** 필터가 너무 제한적이라 결과가 없으면 `필터 초기화` 버튼이 툴팁과 함께 표시됩니다.
- **보안 및 권한 주의:** 라이선스 정책과 개인정보 포함 여부를 구매 전에 반드시 확인해야 하며, 다운로드 링크는 72시간 후 만료됩니다.
- **관련 링크:** 데이터셋 상세, 파인튜닝 마법사, 청구 내역.
- ![스크린샷: 데이터셋 마켓](path/to/image.png "필터 및 카드 목록")

### 3.5 데이터셋 상세
- **목적:** 데이터셋의 구조와 품질을 검토하고 다운로드 또는 구매 결정.
- **접근 경로:** 데이터셋 목록 카드 클릭, 즐겨찾기, URL(`/datasets/{id}`).
- **주요 UI 구성:** 데이터셋 개요 헤더, 메타데이터 표(포맷, 크기, 샘플 수, 업데이트 날짜), 샘플 프리뷰, 가격과 라이선스 정책, 다운로드 버튼.
- **사용 절차:**
  1. 메타데이터 표를 검토한다 → 프로젝트 요구사항과 일치 여부 판단 → _예시:_ JSONL 포맷, 5GB, 100k 샘플.
  2. `샘플 다운로드`로 일부 데이터를 확인한다 → 품질 검증 후 `구매하기`로 이동 → _예시:_ CSV 샘플을 내려받아 열 구조 확인.
  3. 결제가 완료되면 `다운로드 링크 발급` 버튼이 활성화되고, 링크를 클릭해 데이터를 받는다.
- **권장 사용 패턴:** `변경 이력` 탭에서 데이터 업데이트 주기를 확인하고, 파인튜닝 전에 데이터 전처리 가이드를 참고합니다.
- **오류 및 해결:** 다운로드 링크가 만료되면 `새 링크 요청` 버튼을 통해 24시간 유효한 새 링크를 발급받습니다.
- **보안 및 권한 주의:** 개인정보 포함 데이터는 POLICIES에 따라 익명화 여부를 확인하고, 다운로드 후 내부 보안 정책에 맞게 저장합니다.
- **관련 링크:** 파인튜닝 마법사 데이터 선택, 청구 영수증, 라이선스 정책.
- ![스크린샷: 데이터셋 상세](path/to/image.png "메타데이터 및 다운로드")

### 3.6 플레이그라운드
- **목적:** 브라우저에서 즉시 모델을 테스트하고 응답 품질과 비용을 확인.
- **접근 경로:** 사이드바 `플레이그라운드`, 모델 상세의 `플레이그라운드에서 테스트`, URL(`/playground`).
- **주요 UI 구성:** 프리셋 선택 드롭다운, 모델 선택 드롭다운, 시스템 프롬프트 입력, 파라미터 슬라이더(Temperature, Max Tokens, Top-P), 대화 로그, 비용 추적 패널, 결과 내보내기 버튼(cURL/JavaScript/Python).
- **사용 절차:**
  1. 프리셋과 모델을 선택하고 시스템 프롬프트를 설정한다 → 채팅 환경이 맞춤 구성 → _예시:_ `번역` 프리셋 + `Summarizer Pro` 모델.
  2. 사용자 메시지를 입력하고 `보내기` 아이콘을 클릭한다 → 응답과 비용이 대화 로그에 실시간으로 표시 → _예시:_ "이 문장을 영어로 번역" 입력.
  3. `결과 내보내기`를 통해 동일 설정을 코드 스니펫으로 복사해 개발 환경에서 재현한다.
- **권장 사용 패턴:** 응답 품질 비교를 위해 `세션 복제` 기능으로 동일 프롬프트를 다른 모델과 나란히 테스트합니다.
- **오류 및 해결:** API 호출 실패 시 대화 로그에 오류 코드가 표시되며, `다시 시도` 버튼으로 동일 요청을 재전송할 수 있습니다.
- **보안 및 권한 주의:** 프롬프트 내용은 암호화되어 전송되지만 민감한 개인정보는 입력하지 않는 것이 권장됩니다.
- **관련 링크:** 모델 상세, 청구 사용량, 세션 로그 다운로드.

### 3.7 크리에이터 허브
- **목적:** 내가 등록한 모델과 데이터셋의 성과를 추적하고 업데이트 관리.
- **접근 경로:** 헤더 `크리에이터`, URL(`/creator`).
- **주요 UI 구성:** 요약 카드(판매, 다운로드, 총 수익), 리소스 목록, 검수 상태 필터, 액션 버튼(편집, 비공개, 업데이트 제출), 고객 피드백 섹션.
- **사용 절차:**
  1. `새 모델 등록` 혹은 `새 데이터셋 등록`을 클릭해 제작 흐름을 시작한다.
  2. 기존 리소스를 선택하여 가격을 조정하거나 비공개로 전환한다 → 변경 사항이 즉시 반영되고 감사 로그에 기록.
  3. 고객 피드백을 검토하고, `응답 작성`을 통해 문의에 답변한다 → 응답 내용은 공개 Q&A에 표시.
- **권장 사용 패턴:** 월별 판매 보고서를 다운로드해 수익 추세를 분석하고, 릴리스 노트를 상세하게 작성해 고객 신뢰를 높입니다.
- **오류 및 해결:** API 응답이 없으면 상단 알림 배너에서 상태를 확인하고, `새로고침`으로 데이터를 다시 요청합니다.
- **보안 및 권한 주의:** 팀 계정이 아닌 개인 계정으로 로그인했을 경우, 협업 권한을 별도로 초대해야 합니다.
- **관련 링크:** 모델 등록, 데이터셋 등록, 정산서 다운로드.

### 3.8 파인튜닝 마법사
- **목적:** 사용자 데이터로 모델을 재학습하고 결과 모델을 배포 또는 판매에 활용.
- **접근 경로:** 사이드바 `파인튜닝`, 모델 상세의 `파인튜닝 시작`, URL(`/finetune`).
- **주요 UI 구성:** 단계별 진행 표시(데이터 선택 → 기본 설정 → 고급 설정 → 학습 실행 → 결과 리뷰), 로그 뷰어, 진행률 차트, 예상 비용 패널, 리소스 사용량 그래프.
- **사용 절차:**
  1. 데이터셋을 업로드하거나 마켓에서 선택한다 → 데이터 무결성 검사가 실행되고 예상 비용이 계산 → _예시:_ "Medical Image Dataset" 선택.
  2. 기본 하이퍼파라미터(학습률, 배치 크기, 에폭)를 선택한다 → 권장값 툴팁이 제공 → _예시:_ 학습률 0.0003, 배치 64.
  3. `학습 시작`을 클릭해 실시간 로그와 진행률을 모니터링한다 → 중간에 `일시정지` 또는 `중지` 가능.
  4. 학습이 완료되면 결과 모델을 저장하고, `배포 설정`으로 바로 연결한다.
- **권장 사용 패턴:** 장시간 학습 시 `이메일 알림` 옵션을 활성화하여 완료 알림을 받습니다.
- **오류 및 해결:** 학습 실패 시 로그 뷰어에서 오류 코드를 확인하고, `재시작` 기능으로 마지막 체크포인트부터 재개합니다.
- **보안 및 권한 주의:** 업로드 데이터는 POLICIES에 따른 PII 암호화 규칙을 준수해야 하며, 기업 비밀 데이터는 사전 승인 후 업로드합니다.
- **관련 링크:** 데이터셋 마켓, 학습 결과 배포, 모델 등록.
- ![스크린샷: 파인튜닝 마법사](path/to/image.png "훈련 진행 현황")

## 4. 운영 관리

### 4.1 사용자 및 권한 관리
관리자는 `관리자 콘솔`(URL `/admin`)에서 사용자 리스트를 조회하고 역할을 변경할 수 있습니다.
1. 검색창에서 사용자 이메일을 입력 → 결과 목록이 필터링 → _예시:_ `creator@company.com` 검색.
2. 역할 드롭다운에서 새 역할을 지정 → 즉시 권한이 업데이트 → _예시:_ `일반 사용자` → `크리에이터` 승격.
3. `MFA 초기화` 버튼으로 2단계 인증을 초기화 → 사용자가 재설정 링크를 받음.

### 4.2 데이터 백업 및 복구
- 백업은 매일 새벽 3시에 자동 수행되며, 관리자 콘솔의 `백업` 탭에서 최근 30일 내역을 확인할 수 있습니다.
- 수동 백업이 필요한 경우 `즉시 백업` 버튼을 누르면 S3 호환 스토리지에 스냅샷이 저장되고 다운로드 링크가 생성됩니다.
- 복구는 `복구 요청` 폼을 제출하면 운영팀 승인을 거쳐 진행되며, 복구 중에는 관련 서비스가 일시 읽기 전용으로 전환됩니다.

### 4.3 환경 설정

| 설정 항목 | 기본값 | 선택 가능한 범위 | 적용 효과 |
| --- | --- | --- | --- |
| 세션 타임아웃 | 30분 | 15~120분 | 사용자 비활성 시 로그아웃까지의 시간 |
| 결제 한도 | 1,000 Lamports | 100~10,000 Lamports | 월간 결제 상한 설정 |
| 모델 업로드 제한 | 5GB | 1~20GB | 단일 모델 파일 업로드 제한 |
| 데이터셋 접근 로그 보존 | 90일 | 30~365일 | 감사 로그 보존 기간 |

### 4.4 로그·모니터링·장애 대응
- `실시간 로그` 탭에서 API 응답 시간, 오류율, 결제 상태를 모니터링합니다.
- 장애 발생 시 운영팀 체크리스트:
  1. 상태 페이지 확인 → 전체 서비스 혹은 특정 모듈 장애 여부 파악.
  2. `알림 센터`에서 경고 세부 정보를 읽고 담당자 배정.
  3. 영향 범위 분석 후 `공지 배너` 발송.
  4. 근본 원인을 해결하고 `사후 보고서`를 작성해 Changelog에 기록.

## 5. 통합 및 확장 기능

### 5.1 외부 서비스 연동
- **결제 게이트웨이:** Solana 기반 지갑(Phantom, Solflare)을 지원하며, 결제 전 지갑 서명이 필요합니다.
- **이메일/SMS 알림:** SendGrid, Twilio와 연동하여 온보딩, 결제, 파인튜닝 완료 알림을 발송합니다.
- **팀 협업 툴:** Slack 웹훅을 설정하면 모델 검수 완료 및 파인튜닝 완료 시 팀 채널에 메시지가 전송됩니다.

### 5.2 Webhook 및 API 사양

| 엔드포인트 | 메서드 | 설명 | 주요 파라미터 | 예시 응답 |
| --- | --- | --- | --- | --- |
| `/api/v1/models` | GET | 모델 목록 조회 | `page`, `filter`, `sort` | `{ "items": [], "total": 120 }` |
| `/api/v1/models/{id}` | GET | 모델 상세 조회 | `id` | `{ "name": "Summarizer Pro" }` |
| `/api/v1/orders` | POST | 결제 생성 | `modelId`, `plan`, `walletSignature` | `{ "status": "pending" }` |
| `/api/v1/webhooks/finetune` | POST | 파인튜닝 상태 콜백 | `event`, `modelId`, `status` | `{ "event": "FINISH" }` |

Webhook은 HMAC-SHA256 서명을 통해 검증하며, `X-ModelHub-Signature` 헤더를 확인해야 합니다.

## 6. 보안 및 개인정보 보호

### 6.1 인증 정책
- 비밀번호는 최소 10자, 대문자·소문자·숫자·특수문자를 모두 포함해야 합니다.
- 세션 타임아웃 기본값은 30분이며, 민감 권한(관리자)은 10분으로 자동 단축됩니다.
- 2단계 인증은 이메일 또는 OTP 앱(Google Authenticator) 중 선택 가능합니다.

### 6.2 개인정보 취급 및 삭제
- 개인정보는 지역별 데이터센터에 암호화(AES-256)되어 저장됩니다.
- 사용자는 `계정 설정 > 개인정보`에서 데이터 다운로드와 삭제를 요청할 수 있으며, 삭제 요청은 7일 내 처리됩니다.
- 고객 지원 요청에 포함된 개인정보는 30일 후 자동 파기됩니다.

## 7. 자주 묻는 질문
1. **모델 구매 후 환불이 가능한가요?** → 구매 후 24시간 내 사용량이 10% 이하일 경우 지원팀을 통해 환불 요청이 가능합니다. 관련 링크: [청구 및 사용량 대시보드](#39-청구-및-사용량-대시보드).
2. **파인튜닝 학습 시간이 길어지는 이유는 무엇인가요?** → 데이터 크기와 파라미터 설정에 따라 학습 시간이 달라집니다. `예상 시간` 패널을 확인하고 필요 시 배치 크기를 줄이세요. 관련 링크: [3.8 파인튜닝 마법사](#38-파인튜닝-마법사).
3. **지갑 서명 오류가 발생합니다.** → 지갑 팝업이 차단되었거나 세션이 만료된 경우입니다. 팝업 차단 해제 후 다시 시도하세요. 관련 링크: [2.1 계정 생성과 로그인](#21-계정-생성과-로그인).

## 9. 용어집
- **Lamports:** Solana 네트워크에서 사용되는 최소 단위의 디지털 자산.
- **파인튜닝:** 기존 모델을 새로운 데이터로 재학습하여 성능을 개선하는 과정.
- **온보딩 투어:** 신규 사용자가 핵심 기능을 빠르게 익힐 수 있도록 돕는 인터랙티브 가이드.
- **Webhook:** 외부 시스템에 이벤트 발생을 알리는 HTTP 콜백 메커니즘.
- **MFA(다중 요소 인증):** 비밀번호 외 추가 인증 수단을 요구하는 보안 절차.
